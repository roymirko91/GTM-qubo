## 🎤 SCRIPTS DE DESCUBRIMIENTO (50+ Scripts)

### MEDDPICC - Marco Completo (7 elementos)

**M - MÉTRICAS** (¿Qué define éxito?)
```
Preguntas:
"¿Cuáles son las métricas clave por las que eres responsable este año?"
"Si implementáramos una solución como la nuestra, ¿qué sería éxito?"
"¿Cómo miden actualmente [ÁREA_PROBLEMA]?"
"¿Cuál sería un resultado 'home run'?"
"Repásame tus KPIs - ¿cuál es el más importante?"
"Si mejoráramos [MÉTRICA] en 20%, ¿sería significativo?"
```

**E - ECONOMIC BUYER** (¿Quién controla el presupuesto?)
```
Preguntas:
"¿Quién tiene la palabra final en una decisión de compra como esta?"
"¿Reporta a vos o hay alguien arriba?"
"En tu proceso, ¿quién tiene autoridad de aprobación para $[MONTO]?"
"Explicame el proceso de aprobación para inversiones en [DEPARTAMENTO]"
"Más allá de vos, ¿quién más necesita dar el OK?"
"Si miramos una inversión de $[MONTO], ¿quién decide?"
```

**D - DECISION CRITERIA** (¿Qué evalúan?)
```
Preguntas:
"¿Qué evaluarías en una solución como la nuestra?"
"Si te mostrara top 5 cosas a buscar en este espacio, ¿qué incluirías?"
"¿Qué son must-haves vs nice-to-haves?"
"Recorre tus criterios de evaluación"
"¿Qué se vería como 'best-in-class' para vos?"
"¿Qué necesitaría ser verdad para implementar algo en 90 días?"
```

**D - DECISION PROCESS** (¿Cómo compran?)
```
Preguntas:
"Recorre tu proceso de compra. ¿Cómo evalúan típicamente soluciones?"
"¿Cuál es tu timeline de evaluación?"
"¿Hay pasos formales (RFP, vendor evaluation) o más flexible?"
"¿Cuántos vendors típicamente evalúan?"
"¿Cuánto toma una evaluación típica?"
"¿Qué sigue después del demo - cuál es el próximo paso?"
"¿Han hecho esto antes? ¿Cuánto tomó?"
```

**P - IDENTIFY PAIN** (¿Qué problema resuelven?)
```
Preguntas:
"¿Qué está impulsando esta iniciativa?"
"Cuéntame sobre [ÁREA_PROBLEMA]. ¿Cómo impacta tu negocio?"
"¿Cuál es el costo de NO resolver esto?"
"¿Cómo está afectando esto a [TEAM/REVENUE/EFFICIENCY]?"
"¿Qué han intentado hasta ahora?"
"Si no resuelven esto en [6/12] meses, ¿qué pasa?"
"Ayúdame a entender - ¿cuál es el business impact?"
```

**I - IMPENDING EVENT** (¿Cuál es el timeline?)
```
Preguntas:
"¿Cuándo necesitan tener esto resuelto?"
"¿Hay un evento/deadline específico impulsando esto?"
"¿Cuándo querrían tener algo implementado?"
"¿Hay ciclos de presupuesto del que deberíamos estar enterados?"
"¿Cuándo termina tu fiscal year?"
"¿Hay un deadline de proyecto o evento de negocio creando urgencia?"
"¿Cuál es tu target go-live?"
```

**C - CHAMPION** (¿Quién es tu aliado interno?)
```
Preguntas:
"¿Si esto fuera perfecto, serías el main point of contact?"
"¿Quién más de tu equipo debería estar en la evaluación?"
"¿Quién sería el main user de una solución como esta?"
"¿Hay alguien en tu equipo particularmente passionate sobre resolver esto?"
"¿Quién necesitaría ser convencido internamente?"
"¿Serías el project sponsor o sería otra persona?"
```

## 🛑 MANEJO DE OBJECIONES (50+ Objeciones)

### Framework de 5 Pasos

```
1. ACKNOWLEDGE (Muestra que escuchaste)
   "Ese es un gran punto. [Objeción] es definitivamente importante."

2. CLARIFY (Asegúrate de entender)
   "Ayúdame a entender - ¿la preocupación es [X] o [Y]?"

3. REFRAME (Muestra diferente perspectiva)
   "Lo que típicamente vemos es..."
   "La mayoría de equipos en tu situación..."

4. PROOF (Proporciona evidencia)
   "[CLIENTE] tenía la misma preocupación. Aquí está lo que pasó..."
   "Aquí están los números..."

5. CLOSE (Consigue compromiso al próximo paso)
   "¿Eso direcciona tu preocupación? ¿Podemos avanzar con [PRÓXIMO_PASO]?"
```

---

### OBJECIONES DE PRECIO (3-5 respuestas cada una)

#### "Tu precio es muy alto"

**Opción 1: Precio vs. Valor**
```
ACKNOWLEDGE: "Entiendo. Precio es importante, especialmente para una solución nueva."
CLARIFY: "¿La preocupación es la inversión anual o el ROI?"
REFRAME: "Lo que típicamente ven los teams es que la inversión se recupera en 3 meses..."
PROOF: "[CLIENTE] pensaba lo mismo. Al final vieron $X en ahorros en 90 días."
CLOSE: "¿Podemos hacer una evaluación de 30 días para que veas el valor?"
```

**Opción 2: Precio es un Ancla**
```
CLARIFY: "Si precio no fuera un factor, ¿avanzarías? ¿O hay otras preocupaciones?"
[Escucha - frecuentemente precio NO es el problema real]
REFRAME: "La pregunta no es 'es caro', sino 'es valioso'"
PROOF: "Aquí está el análisis de ROI..."
ALTERNATIVE: "También tenemos opción scaled-down que da 80% de valor por 50% del costo"
```

**Opción 3: Precio Competitivo**
```
ACKNOWLEDGE: "[COMPETITOR] tiene pricing agresivo, verdad"
CLARIFY: "¿Estamos comparando apples-to-apples? Déjame mostrar qué está incluido..."
REFRAME: "La mayoría elige la opción más barata. Después de 6 meses se dan cuenta que pagaban por features que no usaban..."
PROOF: "[CUSTOMER] cambió FROM [COMPETITOR] TO nosotros. Aquí está por qué..."
CLOSE: "¿Hacemos un analysis lado-a-lado de ROI?"
```

#### "Ya estamos usando [COMPETIDOR]" (3-5 respuestas)

**Opción 1: Caso de Uso Diferente**
```
"[COMPETIDOR] es sólido en X. Pero ¿cómo manejan [DIFERENTE_PROBLEMA]?
Ese es típicamente donde companies encuentran gaps con [COMPETIDOR].
Aquí está lo que vemos en [CUSTOMER]..."
```

**Opción 2: Complementario**
```
"[COMPETIDOR] es bueno para X. Pero companies todavía necesitan solución diferente para Y.
Actuálmente trabajamos al lado de [COMPETIDOR] en la mayoría de accounts.
La razón: [CUSTOMER_EXAMPLE]..."
```

**Opción 3: Mejor TCO**
```
"Déjame preguntarte - ¿cómo está [COMPETIDOR] handles [FEATURE_WE_OWN]?
La mayoría de implementaciones de [COMPETIDOR] terminan pagando más en workarounds.
Aquí está el análisis de costo total..."
```

---

### OBJECIONES DE TIMELINE (2-3 respuestas cada una)

#### "No estamos listos todavía"

**Opción 1: Buying Signal**
```
"Entiendo. Timing lo es todo.
¿Qué significa 'listo'? ¿Es [TIMELINE/RESOURCE/BUDGET] relacionado?
¿O es más 'no estoy seguro del valor'?"
[Direcciona la VERDADERA objeción]
```

**Opción 2: Crear Urgencia**
```
"Interesante. ¿Qué pasa si no resuelven [PROBLEMA] en los próximos 6 meses?
¿Cuál es el costo del delay?"
[Espera respuesta - revela el pain real]
REFRAME: "Lo que ven la mayoría es que cuando esperan, el costo de delay es mayor que la inversión..."
PROOF: "[CUSTOMER] también pensó tenían tiempo. Esperaron 6 meses y perdieron [NEGATIVE_OUTCOME]."
ALTERNATIVE: "¿Hacemos una evaluación rápida ahora para que estés listo cuando necesites?"
```

---

### OBJECIONES DE PRESUPUESTO (2-3 respuestas)

#### "No tenemos presupuesto"

**Opción 1: ¿Es Realmente Presupuesto?**
```
CLARIFY: "¿Presupuesto está completamente asignado o hay flexibility para high-ROI investments?"
REFRAME: "Lo que típicamente encuentro es que presupuesto no es el constraint real - es claridad de ROI."
PROOF: "Una vez que mostramos el retorno, presupuesto magicamente aparece..."
ALTERNATIVE: "¿Podemos analizar el ROI para que tengas el case cuando presupuesto libere?"
```

**Opción 2: Create Business Case**
```
"Entiendo presupuesto está apretado.
Los CFOs smart guardan 10-15% flexible budget para oportunidades que cumplan el hurdle.
Esta cumple. Aquí está la ROI real: [NÚMEROS]"
ALTERNATIVE: "¿Trabajamos con tu finance team para hacer el business case?"
```

## 🚀 WORKFLOWS COMPLETOS

### WORKFLOW: COLD TO CLOSE (8 etapas)

```
1. RESEARCH (Investigación)
   - Identifica target accounts
   - Encuentra buying signals
   - Researcha decision makers
   - Prepara talking points

2. OUTREACH (Prospección)
   - Email frío personalizado
   - Follow-up en 3-7 días
   - LinkedIn connection + mensaje
   - Second follow-up
   - Transition a phone call

3. DISCOVERY (Descubrimiento)
   - Qualification call (MEDDPICC)
   - Uncover pain, metrics, timeline
   - Build champion
   - Decide si continuar

4. DEMO (Demostración)
   - Demo tailored a su use case
   - Show ROI specific
   - Handle objections
   - Proposal next step

5. PROPOSAL (Propuesta)
   - Business case + ROI
   - Pricing options
   - Implementation timeline
   - Contract terms

6. NEGOTIATION (Negociación)
   - Price discussion
   - Contract review
   - Security/legal questions
   - Executive approval

7. LEGAL/PROCUREMENT (Procura)
   - Legal review
   - Security review
   - RFP response (if needed)
   - Paperwork execution

8. CLOSE (Cierre)
   - Signed contract
   - Kickoff meeting scheduled
   - Implementation plan ready
   - Customer success assigned
```

**Timeline Típico:** 2-6 meses  
**Deal Value:** $50K-$500K+  
**Win Rate:** 20-30%

## 🎓 MÉTRICAS CLAVE POR ROL

### SDR Metrics
- Activity: Emails, calls, meetings booked
- Response rate: % que responden
- Meeting set rate: % que se convierten a meeting
- Pipeline value generated: $ de pipeline creado

### AE Metrics
- Pipeline coverage: Pipeline / Quota (3-4x typical)
- Win rate: % de deals que cierran
- Sales cycle: Días de lead a closed deal
- Average deal size: $ promedio por deal

### CSM Metrics
- Net Revenue Retention (NRR)
- Churn rate: % de customers que se van
- Expansion revenue: Upsell + cross-sell
- Health score: % de customers healthy

### Manager Metrics
- Team activity: Output total del equipo
- Pipeline: $ en cada stage
- Forecast accuracy: % acertado
- Team coaching: Quality of interactions
